### 📋 Flutter UI 转化与开发规范 (UI Migration Rules)

#### 🎯 1. 核心原则 (Core Principles)
*   **高保真还原 (Pixel-Perfect)**: 严格对照原 Web 端 (Vue3) 项目的设计细节，视觉还原度需达到 95% 以上。包含色彩、排版、阴影层级、组件圆角及页面过渡动画。
*   **单一数据源 (Source of Truth)**: 原 Web 侧代码（CSS/SCSS 变量、交互逻辑）作为视觉和交互的唯一参考基准。
*   **零溢出容忍 (Zero Overflow)**: 确保在多尺寸设备（iPhone、iPad、Android 主流机型）及极端文本长度下，UI 布局自适应且无任何 `RenderFlex overflowed` 报错。
*   **动静分离**: 现阶段仅进行静态 UI 和交互还原。必须为后续 API 接入预留标准入参，并妥善处理 UI 的边界状态（Empty State 空状态、Loading 骨架屏、Error 状态）。

#### 🏗️ 2. 架构与技术栈约束 (Tech Stack Constraints)
*   **核心环境**: 强制使用 Flutter `>=3.19.0` 与 Dart `>=3.3.0`。
*   **路由管理 (`go_router`)**: 统一使用声明式路由。必须配置嵌套路由，并通过自定义 `PageTransitionsBuilder` 还原 Web 端的页面切换动画（如 Fade 淡入淡出、Slide 滑动切换）。
*   **状态管理**: 仅使用 `provider` 或 `flutter_hooks` 管理 UI 内部的精细交互状态（如复杂动画控制、多步骤表单状态、展开/折叠），严禁滥用全局状态。
*   **资源加载**: 
    *   矢量图形强制使用 `flutter_svg`，保证视网膜屏幕高清渲染。
    *   必须通过静态类统一管理资源路径（如封装 `AppImages.logo`），**严禁**在业务代码中出现 `"assets/images/logo.png"` 等硬编码字符串。

#### 🎨 3. UI 与样式规范 (Styling & Theming)
*   **禁止硬编码颜色/尺寸**: 严禁在组件中直接写死颜色（如 `Color(0xFF...)`）。必须通过 `ThemeData` 和 `ThemeExtension` 获取全局设计令牌（Design Tokens），如 `Theme.of(context).extension<AppColors>()`。
*   **尺寸与屏幕适配 (`flutter_screenutil`)**: 强制使用等比例缩放。所有宽高、边距、圆角、字号必须使用扩展方法（`.w`, `.h`, `.r`, `.sp`）。
    *   *例外*: 极细的边框线（如 `1px` 分割线）通常固定为 `1.0` 或 `0.5`，不参与缩放。
*   **排版系统对齐**:
    *   严格映射 Web 端的 `font-size`, `font-weight`, `line-height`。
    *   **注意**: Flutter 的 `height` 属性为行高与字号的比值（`line-height / font-size`），需精确换算。
    *   必须配置 `fontFamilyFallback` 保证多语言下的基线对齐。
*   **视觉修饰还原**:
    *   **阴影**: CSS `box-shadow` 需转化为 Flutter `BoxShadow`，精确匹配 `color`, `offset`, `blurRadius` (模糊半径) 和 `spreadRadius` (扩散半径)。
    *   **毛玻璃**: CSS `backdrop-filter: blur` 需使用 `BackdropFilter` 配合 `ImageFilter.blur` 实现。

#### 🧩 4. 组件化与布局规范 (Components & Layout)
*   **自底向上开发**: 优先封装高频复用的原子组件（如 `CustomButton`, `CustomTextField`, `CustomNavBar`），再进行页面级复合拼装。
*   **交互态与微动画**: 按钮和卡片的 Hover、Active、Focus 状态必须有视觉反馈。使用 `MaterialStateProperty` 或 `AnimatedContainer`、`AnimatedOpacity` 替代 Web 的 CSS `transition` 平滑过渡颜色和阴影。
*   **弹性布局映射**: 
    *   对应 Web `Flexbox`：使用 `Row`/`Column`，合理配置 `mainAxisAlignment` / `crossAxisAlignment`，配合 `Expanded` 处理自适应。
    *   对应 Web `Grid`：使用 `SliverGrid` 或 `GridView.builder` 配合 `crossAxisCount` 和 `childAspectRatio` 精确还原。
*   **安全区域包裹**: 针对刘海屏和底部小白条，全局合理使用 `SafeArea`，或利用 `MediaQuery.padding` 调整 `Padding`，防止内容被系统 UI 遮挡。

#### 📁 5. 目录与命名规范 (Directory & Naming)
*   **文件命名**: 严格遵循 Dart 的 `snake_case`（下划线命名法），如 `home_screen.dart`, `app_colors.dart`。
*   **类命名**: 严格遵循 `PascalCase`（大驼峰命名法），如 `HomeScreen`, `CustomButton`。
*   **结构划分**:
    *   `lib/theme/`: 存放设计令牌（颜色、排版、全局主题映射）。
    *   `lib/widgets/`: 存放跨页面高频复用的原子级组件。
    *   `lib/screens/`: 按业务模块划分的完整页面。
    *   `lib/router/`: 存放路由表与转场动画配置。
