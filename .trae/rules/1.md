## **2. 技术栈与架构约定 (Flutter SDK 3.11+)**

### **状态管理 (State Management)**
- **强制要求**：使用 `flutter_riverpod` (Notifier/AsyncNotifier)。
- **禁止使用**：`ChangeNotifier`。
- **Provider 声明**：
  - 优先使用 `.autoDispose` 配合 `.new` 语法。
  - 示例：`final myProvider = NotifierProvider.autoDispose<MyNotifier, MyState>(MyNotifier.new);`

### **网络通信 (Networking)**
- **API 实例**：使用基于 `Dio` 封装的全局 API 实例。
- **响应处理**：所有 API 响应必须映射到 `ApiResponse<T>`，并严格处理 `isSuccess`、`code` 和 `msg`。
- **数据模型**：严禁使用 `dynamic`，必须为所有 Request/Response 定义清晰的 `json_serializable` 模型。

### **路由导航 (Routing)**
- **工具**：使用 `go_router` 声明式路由。

### **代码生成 (Code Generation)**
- **工具**：`json_serializable` & `build_runner`。
- **执行命令**：修改模型后，必须运行 `dart run build_runner build --delete-conflicting-outputs`。

### **目录规范 (Directory Structure)**
- `lib/models/`：数据模型定义。
- `lib/providers/`：Riverpod 状态管理。
- `lib/services/`：API 调用及底层业务逻辑。
- `lib/screens/`：完整页面 UI。
- `lib/widgets/`：可复用的 UI 组件。
- `lib/theme/`：全局样式定义。

### **Web 适配 (Web Optimization)**
- **图片加载**：所有网络图片必须使用 `WebSafeImage` 包装，以处理跨域和内存问题。
- **交互适配**：支付、弹窗等交互需考虑 Web 平台特性。

### **UI 与 异常处理 (UI & Error Handling)**
- **样式统一**：使用 `lib/theme/app_theme.dart` 中的定义。
- **异常反馈**：API 调用必须包含 `isLoading` 状态、`try-catch` 捕获以及 `ToastUtils` 反馈。
